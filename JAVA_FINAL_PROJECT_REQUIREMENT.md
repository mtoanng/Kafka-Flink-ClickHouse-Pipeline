# TÓM TẮT HƯỚNG DẪN THỰC HIỆN ĐỒ ÁN JAVA 2026

## 1. Thông tin chung và Quy định nhóm

* 
**Lập nhóm:** Tối đa 5 sinh viên/nhóm, có 1 nhóm trưởng; không chấp nhận làm đồ án cá nhân.


* 
**Yêu cầu đề tài:** Thiết kế và phát triển ứng dụng Java có kết nối cơ sở dữ liệu (CSDL), sử dụng mô hình 3 lớp hoặc MVC.


* **Quản lý dự án:**
* Phải có trang giới thiệu nhóm (Wordpress, Google, Notion, GitHub Pages...) để tóm lược mục tiêu, danh sách thành viên, tiến độ tuần, thiết kế giao diện, backlog, video giới thiệu....


* Sử dụng Repository chung (GitHub, Bitbucket...) để quản lý mã nguồn, thực hiện Pull Request (PR) và Code Review giữa các thành viên.





## 2. Sản phẩm bàn giao (Deliverables)

Nhóm cần nộp đầy đủ các thành phần sau trên hệ thống courses vào ngày báo cáo cuối kỳ:

1. 
**Quyển báo cáo:** Định dạng PDF/DOCX theo mẫu của trường, trích dẫn chuẩn APA/IEEE/Harvard.


2. 
**File thuyết trình:** PowerPoint bao gồm giới thiệu, quản lý dự án, công cụ, thiết kế, kiến trúc, demo và Q&A .


3. 
**Video Demo:** Minh họa hoạt động của sản phẩm.


4. 
**Source Code:** File Zip chứa toàn bộ mã nguồn (loại bỏ thư mục rác như /bin, /target), file script database và file README hướng dẫn cài đặt.



## 3. Yêu cầu kỹ thuật và Chức năng

* 
**Công nghệ cốt lõi:** Java Core (OOP, Swing/JavaFX, JDBC); sử dụng Maven hoặc Gradle để quản lý thư viện.


* 
**Kiến trúc:** Áp dụng lập trình hướng đối tượng (OOP), độc lập CSDL, tách biệt giao diện và nghiệp vụ (3 lớp/MVC).


* **Chức năng bắt buộc:**
* Đăng ký/Đăng nhập, phân quyền người dùng (End-user, Quản lý, Admin).


* Các chức năng CRUD (Thêm, Xóa, Sửa, Đọc).


* Tìm kiếm đa tiêu chí, thống kê và xuất báo cáo (Report) ra file.


* Kiểm tra tính hợp lệ của dữ liệu nhập (Validation) và chống tấn công.




* 
**Nâng cao:** Viết Unit Test bằng JUnit, áp dụng các mẫu thiết kế (Design Patterns), và có các sáng tạo triển khai thực tế.

## 5. Chính sách chống đạo văn và Sử dụng AI

* 
**Tỷ lệ tham khảo:** Không quá 15% tổng lượng văn bản và mã nguồn từ các đồ án khác; mọi tham khảo phải có trích dẫn.


* 
**Sử dụng AI:** Được phép sử dụng công cụ AI để hỗ trợ nhưng phải mô tả rõ loại AI, cách dùng, nội dung AI cung cấp và cách nhóm kiểm tra lại.


* 
**Nguyên tắc:** Nhóm phải hiểu và giải thích được mọi thông tin/mã nguồn đã nộp.
